windows_nginx_tomcat负载均衡小搭建

发表于 2016-03-31   |   分类于 服务器配置   |     |   评论   |   阅读

在windows系统中使用nginx，tomcat进行负载均衡搭建


1. 环境准备：

  nginx：nginx-1.8.1


  tomcat：tomcat-8.0.28 两个


  windows：win8

2. 修改其中一个tomcat的端口


  打开tomcat目录下的\conf\server.xml。修改三处：


  第一处：

<Server port="18006" shutdown="SHUTDOWN">

  第二处：

<Connector port="18081" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443" />

  第三处：

<Connector port="18010" protocol="AJP/1.3" redirectPort="8443" />

  这三处端口号需与另一个tomcat配置的端口号不同，同时也要注意配置的端口是否已被其他应用占用。

3. 配置nginx


  打开nginx目录下的\conf\nginx.conf，修改两处：


  第一处：

#gzip on;
upstream my_jq{
    server localhost:8080;
    server localhost:18081;
}

  第二处：

location / {
        root   html;
        index  index.html index.htm;

        #此处的 http://localhost与upstream localhost对应  
        proxy_pass  http://my_jq;  

        proxy_redirect off;  
        proxy_set_header Host $host;  
        proxy_set_header X-Real-IP $remote_addr;  
        proxy_set_header  X-Forwarded-For $proxy_add_x_forwarded_for;  
        client_max_body_size   10m;   
        client_body_buffer_size  128k;  
        proxy_connect_timeout  100;  
        proxy_send_timeout   100;  
        proxy_read_timeout 100;  
        proxy_buffer_size 4k;  
        proxy_buffers  4 32k;  
        proxy_busy_buffers_size 64k;  
        proxy_temp_file_write_size  64k;  
    }

4. 添加测试文件index.jsp


在tomcat目录下\webapps\ROOT\ 新建一个index.jsp。

<!DOCTYPE html>
<html lang="en">
<head>
    <title><%=request.getServletContext().getServerInfo() %></title>
</head>

<body>
   <%
   out.println("this is tomcat_1"); 
   out.println("request.getLocalAddr(): " + request.getLocalAddr());  
   out.println("<br/>");  
   out.println("request.getLocalPort(): " + request.getLocalPort());  
   out.println("<br/>");  
   out.println("Session ID: " + session.getId());  
   out.println("<br/>"); 
   %>  
</body>
</html>

在另一个tomcat也同样新建index.jsp，稍微修改一处：

out.println("this is tomcat_2");

5. 启动服务


  双击nginx，启动nginx服务。进入tomcat下的bin目录，双击startup.bat,启动tomcat服务。


  注意：如果双击startup.dat后一闪而过，没什么反应开启不了服务，可能需要配置文件setclasspath.bat(配置java环境变量)：

#添加下面两行 JAVA_HOME,JRE_HOME
set JAVA_HOME=C:\Program Files\Java\jdk1.8.0_73
set JRE_HOME=C:\Program Files\Java\jre1.8.0_73
rem Make sure prerequisite environment variables are set

rem In debug mode we need a real JDK (JAVA_HOME)
if ""%1"" == ""debug"" goto needJavaHome

  浏览器输入 http:\localhost:80\index.jsp, 多刷新几次会出现不同信息。当关掉其中一个tomcat服务后，仍然可以访问。


  测试成功。
