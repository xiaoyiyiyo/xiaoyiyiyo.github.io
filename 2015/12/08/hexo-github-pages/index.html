hexo-github pages

发表于 2015-12-08   |   分类于 工具   |     |   评论   |   阅读

快速搭建属于你自己的定制博客

  如果你想拥有自己的博客，想自己掌控博客样式，且有那么点耐心想搭建的，那么可以看看这个博客。而不弄博客的，那就凑个热闹吧，鼓鼓掌，拍拍砖就行了，嘿嘿。 搭建基于github pages的博客教程网上有很多，但是每个教程都或多或少不如人意，出现各种问题，遂将我的搭建过程详细写于该博客，作为我博客的开门之作！ ^_^  如有问题,可以留下评论。

Quick Start


创建github账号以及项目

地址 ： https://github.com/

注册登录进去后，点击 + New repository   之后在Repository name的输入框中输入： yourname.github.io (yourname替换成你自己想取的名字)，其他选项等默认即可，点击create repository即完成。先不要关掉github，因为还要用到，嘿嘿。


本地安装git，nodejs

注意: git，nodejs版本的不同可能会造成各种问题。在此给出我的版本。

git: https://git-for-windows.github.io/ ( 注意是windows的，我原先是在 http://git-scm.com/download/这个下载的，结果就坑人了)
nodejs: https://nodejs.org/en/ ( 注意下载 LTS 版本的 )

安装都选默认即可。安装完后，记得进入nodejs的目录，双击运行 nodevars.bat 文件,即完成环境路径配置。当然也可以手动配置。



生成SSH,建立与github的联系

关于SSH的作用有兴趣的可以自行百度，我目前就当它是验证连接，一个公钥(放在github，相当于门)，一个私钥(放在本地,相当于门的钥匙)。

在任意地方右键，会有  Git Bash Here 一栏，点击进入后，先初始化git，输入命令：

$ git config --global user.name "John Doe"    //你的名字，此处为了方便，你可以和github中的名字一样，比如xiaoyiyiyo
$ git config --global user.email johndoe@example.com   //你的邮箱，同样为了方便，和github注册的邮箱一样吧。

下面进行SSH的配置，输入命令：

$ ssh-keygen -t rsa -C "xiaoyiyiyo@outlook.com"

此处直接enter，然后会提示 Enter passphrase 〈empty for no passphrase〉: 即输入你的 密码，这个密码你得记住哦。之后最终会显示出一段你看不懂的类似乱码的东西，即表示成功。

然后输入命令：

$ cat ~/.ssh/id_rsa.pub

这个命令是得到公钥内容，复制屏幕上显示出来的内容。然后打开浏览器进入你的github，

点击图中的 Settings  可以在左边栏中看到 SSH keys  点击之后， Add SSH key  将复制的内容添加进去。

最后测试是否能够连通，输入命令：

$ ssh -T git@github.com

可能会出现要你输入yes/no，果断yes。再之后可能要出现类似

Enter passphrase for key '/c/Users/kingyi/.ssh/id_rsa'

即输入你先前的密码，最终出现

Hi xiaoyiyiyo! You've successfully authenticated, but GitHub does not provide shell access.



安装配置 HEXO ,最后一步了!

首先找一个你想要将hexo安装到的目录，然后在进入目录，右键， Git Bash Here ，然后输入命令：

$ npm install -g hexo

输入完后可能会有一段时间没动静，不要急，等！（哥当时等了好久结果报了个错，没有安装成功，后来找问题，找的我直接换了个nodejs版本，就OK了，醉的我。。。）

然后部署hexo（可以先观察目录结构，了解前后发生的事情），输入命令：

$ hexo init

之后会出现提示：

info You are almost done! Don't forget to run 'npm install' before    ...

呃。。果断输入:

$ npm install  //这个命令应该是在安装hexo需要的依赖模块，见node_modules文件夹

hexo会产生一些博客必要的来源文件，而为了产生真正构建博客的文件，输入命令：

$ hexo g  //即hexo generate

记住这个命令，弄博客必备命令。

至此，hexo会自动在目录中生成网站博客需要的文件，比如 public 文件夹，它是主要根据 source/_posts/hello.md 产生的。

精彩来了，如果你等不及了，想看下你的博客效果，输入命令：

$ hexo s   //即 hexo server

然后在网站输入localhost:4000，按下Enter,出现标准的界面。（呃。。图片没有截完整，看你自己的就行）

嗯嗯，这个是本地的，还需再加油一把，快要成功了！

接下来配置hexo ，在目录中找到_config.yml（根目录哦，因为themes文件夹中也有个），用编辑器打开，转到最后，会看见：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
type:

修改如下：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
type: git
repository: git@github.com:xiaoyiyiyo/xiaoyiyiyo.github.io.git
branch: master

其中 repository: 后面的值为github你相应项目中的值，如下  SSH  ：

当然你也可以输入 HTTPS  中的值。至于哪种方式好，呃，，我选的是SSH，因为我也不清楚，毕竟前面已经配置了。

注意: 配置的时候要有一个空格，比如  type: git  冒号后面有个空格。这个东东坑我了好半天。！！

坚持，，最后两三步，两三个命令了。。。

配置好_config.yml的必要部分后，你可以继续配置一些信息，看文件里的注释就应该知道了。下面就是将目录下的博客文件部署到github里的时候了，先输入一个命令，不然又得坑人（当时我啊。。心酸啊）：

$ npm install hexo-deployer-git --save   //解决下面命令中 hexo d 的报错

百度上说是因为在Hexo 3.0版本后deploy git 被分开的。。。。

然后输入命令：

$ hexo clean   //即用于清除缓存，删除原来生成的文件，如public文件夹里的东西。[可选命令]
$ hexo g   //这个命令上面已介绍
$ hexo d   //即 hexo deploy，部署命令，将文件部署到网络github上

注意：这三个命令就是我们本地操作博客常用到系列操作，得记住啊。

输入完命令，可能会弹出一个框框要你输入一个东东，那个东东应该是要你输入密码，输入你的密码即可。

hexo d 这条命令输入完后，有一种报错，其中部分字段如下：

Permission denied (publickey). fatal: Could not read from remote repository.
...
Please make sure you have the correct access rights and the repository exists.

当时为了解决这个问题，煞费苦心啊，以为是SSH配错了，测试 ssh -T git@github.com 结果一切正常，唉，，，最后百度偶然见到一个同样悲催的同学的问答，原来是 git 版本造成的，，所以咯，文章前面就说了注意版本问题。


至此！！！终于到了



浏览器输入 xiaoyiyiyo.github.io 即可看到你的博客了。xiaoyiyiyo.github.io替换成你的github项目名字，对，就是你创建github项目时的名字。
